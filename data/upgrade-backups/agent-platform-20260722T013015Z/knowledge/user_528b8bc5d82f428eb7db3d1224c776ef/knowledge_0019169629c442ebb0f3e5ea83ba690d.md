## 一、项目空间是什么

WorkBuddy 的「项目空间」是一个**面向团队协同的 AI 工作区**：左侧/顶部以「四大模块 Tab」承载日常协作流（看资料、走流程、盯动态、管任务），底部以「项目配置」沉淀可复用的能力（指令、连接器、专家、技能、自动化）。AI 助手在本项目内的身份与行为，由「指令」统一约束。

```
项目空间
├── 四大协同模块（日常视图）
│   ├── 资产 Assets   —— 资料库 / 文档资产
│   ├── 计划 Plan     —— 5 步上手清单
│   ├── 动态 Activity —— 队友工作动态
│   └── 任务 Tasks    —— 团队协同任务
└── 项目配置（能力沉淀）
    ├── 指令 Instructions
    ├── 连接器 Connectors
    ├── 专家 Experts
    ├── 技能 Skills
    └── 自动化 Automation
```

## 二、四大协同模块（顶部 Tab）

### 1. 资产（Assets）
- **定位**：项目的统一资料库，所有文档、图片、素材集中存放，AI 与成员都能随时取用。
- **底层**：由**云盘连接器（netdrive）**承载。本项目的资料库根目录为 `TskiEtQqLMTZ`，实测包含：
  | 名称 | 类型 | 说明 |
  |---|---|---|
  | `welcome` | Markdown | 项目功能地图入口文档（见指令摘要） |
  | `资产库使用指南` | PDF（≈877 KB） | 资产库使用说明 |
  | `系统文件` | 系统 | 项目系统级文件 |
- **怎么用**：打开资产 Tab → 找 `welcome.md` 学习完整功能地图；上传/检索各类文档供 AI 引用。

### 2. 计划（Plan）
- **定位**：引导新成员快速上手的「5 步上手清单」。
- **作用**：把核心动作拆成可勾选的步骤，跟着走一遍即可完成项目的关键初始化与体验（如配置、召唤 Agent、查看模块等）。
- **怎么用**：打开计划 Tab → 按清单逐条完成并标记，进度一目了然。

### 3. 动态（Activity）
- **定位**：团队**工作动态流**。
- **作用**：实时查看队友（及团队 Agent）的工作进展与产出，确认「谁在做什么、做到哪了」，便于协同对齐、避免重复劳动。
- **怎么用**：打开动态 Tab → 浏览时间线式动态；关注关键进展或评论互动。

### 4. 任务（Tasks）
- **定位**：**团队协同任务**管理。
- **作用**：创建、分配、跟踪任务，了解任务的协同方式（负责人、状态、流转）。任务模块底层由**任务连接器（wb-issues）**承载，支持任务详情、状态流转、子任务与附件。
- **怎么用**：打开任务 Tab → 查看当前团队任务列表；新建/认领/推进任务；可在任务上挂载资料库文件或外部链接作为附件。

> 这四大模块即 `welcome.md` 摘要中提示的探索入口，也是日常协作的主界面。

## 三、项目配置（能力沉淀区）

### 1. 指令（Instructions）
- **定位**：定义 AI 助手在本项目中的**身份、职责与回答口径**——相当于项目的「AI 行为契约」。
- **当前配置（实测）**：项目已写入指令，核心内容两条：
  1. **欢迎与导览**：「欢迎来到 WorkBuddy 项目功能！」并指引探索四大模块；提示「在底部输入框召唤团队 Agent，随时问任何问题」。
  2. **对 AI 的指令**：当在本项目内提问时，优先以 **「WorkBuddy 项目功能介绍助手」** 身份回答，并结合资料库 `welcome.md` 内容与项目已有配置进行说明。
- **影响范围**：本项目内所有对话都会先套用这段指令，确保回答风格与口径一致。

### 2. 连接器（Connectors）
- **定位**：把外部系统接入项目空间的「桥」——通过 MCP（Model Context Protocol）连接。
- **当前配置（实测，见 `.mcp.json`）**：
  | 连接器 | 类型 | 作用 |
  |---|---|---|
  | `netdrive` | HTTP MCP | 云盘 / 资料库：承载「资产」模块，提供文件列举、上传、下载、语义检索 |
  | `wb-issues` | HTTP MCP | 任务 / 问题系统：承载「任务」模块，提供任务（todo）增删改查、留言、附件挂载 |
- **扩展方式**：如需接 GitHub / 工蜂 / CNB / Figma 等，可加入对应连接器或技能（见下方「技能」）。

### 3. 专家（Experts）
- **定位**：可随时**召唤的领域 AI 专家**（团队 Agent）。在项目底部输入框唤出，针对特定领域给出专业支持。
- **当前可用（实测，来自技能市场 `codebuddy-plugins-official`）**，按族类举例：
  | 族类 | 代表专家 |
  |---|---|
  | 编程语言专家 | `python-expert`、`golang-expert`、`typescript-expert`、`rust-expert`、`sql-expert`、`ruby-expert`、`rails-expert` |
  | 专项领域专家 | `mcp-expert`、`agent-expert`、`docusaurus-expert`、`flutter-expert`、`service-mesh-expert`、`arm-cortex-expert`、`threat-modeling-expert`、`mermaid-expert` |
  | 流程/协作专家 | `task-decomposition-expert`（任务拆解）、`command-expert`、`team-workload-balancer`（团队负载均衡） |
- **怎么用**：在输入框召唤对应专家（如「召唤 python 专家」「让 mcp 专家帮我看配置」），由其接手专业子任务。

### 4. 技能（Skills）
- **定位**：可被执行的能力插件，让 AI 具备具体工具与流程（文档处理、浏览器、代码平台、自动化等）。
- **当前配置（实测）**：
  - **已启用插件（见 `settings.json`）**：`pdf`、`docx`、`xlsx`、`pptx`、`agent-browser`、`playwright-cli`、`hot-skills`。
  - **本地技能库（见 `skills/` 目录）**：
    | 技能 | 能力 |
    |---|---|
    | `computer-use` | 虚拟桌面交互：截图、鼠标/键盘操作 GUI |
    | `figma-connector` | Figma 设计稿还原、Design Token 提取 |
    | `github-connector` | GitHub 仓库 / PR / Issues |
    | `cnb-connector` | 腾讯 CNB 代码平台集成（旧版脚本） |
    | `gongfeng-connector` | 工蜂（git.woa.com）仓库 / PR |
    | `preview` | 启动 Web 服务并生成可访问预览 URL |
    | `automation-task-manager` | 自动化定时任务管理（见下） |
- **说明**：技能会按对话上下文**自动激活**；`README` 还列出 `cnb-cli-connector`、`github` 等更多集成能力，可按需安装到本项目。

### 5. 自动化（Automation）
- **定位**：用自然语言创建**定时/周期性任务**，让 AI 在指定时间自动执行指令。
- **当前配置（实测，见 `automation-task-manager` 技能）**：
  - 调度脚本：`.codebuddy/skills/automation-task-manager/scripts/scheduler-api.sh`
  - **三种频率类型**：
    | 类型 | 值 | 适用 |
    |---|---|---|
    | 每天执行 | `daily` | 每日 X 点、工作日、每周 X |
    | 按间隔执行 | `interval` | 每 X 分钟 / 每 X 小时 |
    | 单次执行 | `once` | 某具体日期时间只跑一次 |
  - **时间表达式**：6 位 Cron（`秒 分 时 日 月 周`，秒固定为 0），**最小间隔 1 分钟**（拒绝秒级）。
  - **安全约束**：写操作（创建/编辑/删除）必须**「先预览 → 用户确认 → 再执行」**；只读（列表/详情）免确认。单项目最多 **100 个**自动化任务。
  - **示例**：每天 21:00 提醒写日报（`0 0 21 * * *`，daily）；每小时巡检服务状态（`0 0 * * * *`，interval）；明天下午 3 点提醒交周报（once）。

## 五、给你的下一步建议
1. 打开**计划** Tab，把「5 步上手清单」走一遍，体验四大模块。
2. 打开**资产** Tab 读 `welcome.md`（功能地图全文），并把 `资产库使用指南.pdf` 存为常备参考。
3. 想做定时提醒/巡检 → 用**自动化**创建任务（会先给你预览确认）。
4. 遇到专业问题（如 MCP 配置、Python 实现）→ 在输入框**召唤对应专家**。
5. 需要连 GitHub / Figma / 工蜂等 → 在**连接器 / 技能**里补装对应集成。

> 如后续云盘凭证恢复，可再补取 `welcome.md` 全文，进一步对齐细节。
