# 产品碳足迹SaaS白皮书大纲

> 生成时间：2026-06-22 | 基于竞品调研+市场分析成果

---

## 一、执行摘要
- 一句话定位：面向[目标行业]的产品碳工作台
- 核心数据：因子库覆盖量、标准兼容数、典型交付时效
- 差异化：不做全功能LCA平台，做行业深度+供应链协同

## 二、市场背景与痛点
- 政策驱动：CBAM/ESPR、双碳、链主合规
- 市场格局：海外已分层，国内平台产品化早中期
- 目标客户画像与痛点闭环

## 三、产品定位与价值主张
- 行业碳工作台 vs 全功能LCA
- 核心价值：降低门槛、打通链路、可审计可复用

## 四、核心功能模块
| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | BOM/工艺建模 | 结构化碳排放建模 |
| P0 | 因子库 | 内置+自定义，数据质量标注 |
| P0 | Excel批量导入 | 快速上手指引 |
| P0 | 热点分析 | 自动识别碳排热点 |
| P0 | PCF报告生成 | ISO 14067/GHG Protocol兼容 |
| P0 | 版本管理 | 产品迭代追溯 |
| P1 | 供应商数据收集 | 问卷+校验+催收工作流 |
| P1 | 数据质量评分 | Pedigree矩阵或简化版 |
| P1 | 模板复用 | 同类产品快速套用 |
| P2 | ERP/PLM集成 | BOM自动拉取 |
| P2 | 场景对比 | 假设分析 |
| P2 | API平台 | 批量数据交付 |

## 五、技术架构
数据层 → 建模层 → 计算层 → 协同层 → 输出层 → 集成层

## 六、标准合规与数据质量
ISO 14040/44/67、GHG Protocol、PEF、EPD
数据质量：Pedigree评分、审计追踪

## 七、行业应用场景
选1-2行业深度展开（电子电器 / 汽车零部件 / 电池等）

## 八、客户案例与ROI
实施周期、量化收益（效率提升百分比）、客户证言

## 九、路线图
当前边界 → 6-12月演进 → 大版本规划

## 十、定价与服务模式
SaaS按席位/项目/企业 + 定制实施
