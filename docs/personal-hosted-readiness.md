# 个人托管测试版基线与 G1 准入清单

更新日期：2026-07-21  
适用范围：一个实例、一个个人用户、邀请制测试。  
不适用范围：团队协作、多租户 SaaS、公共注册、自动支付和企业能力。

## 1. 产品交付边界

个人托管测试版交付的是一套可安装、可使用、可升级、可备份和可恢复的个人 Agent 工作台。平台负责提供应用代码、安装与升级方法、安全默认值、备份恢复工具、运行诊断和数据导出/删除能力。

部署者负责准备运行机器、HTTPS/网络入口、模型供应商账号与费用、备份保存位置，并按版本说明执行升级。模型供应商、网络搜索和本机 OCR 属于外部依赖；依赖不可用时平台必须明确降级，不能伪装为成功。

首批测试规则：

- 每个实例只服务一个个人用户，不共享数据库、文件目录和模型密钥。
- 关闭公开注册，由部署者创建管理员或发放单次邀请码。
- 对话、知识、产物和模型密钥默认保留在该实例及其配置环境内。
- 文件大小、存储、模型调用和并发均设置上限。
- 采用人工开通、人工支持，不承诺公开服务 SLA。

## 3. 数据分类与生命周期

| 数据类别 | 当前位置 | 敏感级别 | 默认保留 | G1 要求 |
| --- | --- | --- | --- | --- |
| 账号资料与密码散列 | SQLite：`users` | 高 | 账号存在期间 | 可修改、导出基本资料并删除账号 |
| 会话令牌 | SQLite：`sessions` | 高 | 当前默认最长 14 天 | 过期清理、全部撤销、日志不输出令牌 |
| 对话与回答 | SQLite：`threads/messages` | 高 | 用户删除前 | 导出、删除、备份恢复后权限一致 |
| 知识资料元数据与分片 | SQLite：`knowledge_documents/chunks` | 高 | 用户删除前 | 删除记录、分片和原文件保持一致 |
| 知识原文件 | `data/knowledge/` | 高 | 用户删除前 | 受控下载、备份、删除和容量限制 |
| 文件产物 | SQLite + `data/artifacts/` | 高 | 用户删除前 | 受控下载、导出、删除和容量限制 |
| 长期记忆 | SQLite：`memories` | 高 | 用户删除或停用前 | 导出、删除、继续拒绝秘密内容 |
| Run、步骤、事件与审批 | SQLite：`runs/run_*` | 中高 | 与对话一致 | 诊断可用；导出时不泄露密钥和完整工具参数 |
| 回答与引用评价 | SQLite：`run_feedback/citation_feedback_items` | 中 | 与关联 Run 一致 | 可修改、导出和随 Run 删除 |
| 模型与外部服务密钥 | 环境变量或 `.env` | 最高 | 配置有效期间 | 不进入数据库、前端、日志、备份报告和导出包 |
| 应用日志 | 标准输出或轮转日志 | 中高 | 建议 30 天内 | 只记录必要元数据，提供轮转和清理说明 |
| 技能包与资源 | `server/skills/` 及受控资源目录 | 中高 | 用户删除前 | 脚本不执行；校验大小、路径和资源内容边界 |
| 备份 | 部署者指定目录 | 最高 | 建议滚动保留 | 加密保存、限制访问、定期恢复验证和到期清理 |

删除语义在 G1 前必须明确：在线数据删除应覆盖数据库记录和本地文件；备份中的副本按备份保留周期自然过期。平台不得对用户承诺无法验证的“立即从所有备份物理擦除”。

## 4. 本地性能、用量与存储基线

运行命令：

```bash
python3 -m server.evaluate_personal_hosting \
  --database agent_platform.db \
  --knowledge-dir data/knowledge \
  --artifacts-dir data/artifacts
```

2026-07-21 本地结果：

| 指标 | 基线 |
| --- | ---: |
| Run 样本 | 145 |
| 完成 / 失败 | 141 / 4 |
| 完成率 | 97.24% |
| 平均时延 | 7.178 秒 |
| P50 / P95 | 4.723 / 17.496 秒 |
| 工具调用 | 13 次，平均 0.09 次/Run |
| 模型事件 | 141 次，平均 0.972 次/Run |
| 输入 / 输出 Token 估算 | 535 / 11,179 |
| SQLite 文件 | 2,101,248 字节 |
| 知识原文件 | 26 个，117,356 字节 |
| 产物文件 | 11 个，44,225 字节 |

限制：历史 Run 的 Token 估算字段覆盖不完整，当前数值不能代表真实供应商账单。评估器不内置可能变化的模型单价；只有同时传入输入、输出每百万 Token 单价时才计算估算费用。G1 前需用实际供应商账单核对至少 10 个 Run 的估算误差。

报告只读取数值元数据、事件类型、表记录数和目录文件大小，不读取或导出用户 ID、对话正文、知识正文、产物内容或密钥。

## 5. 风险登记

| ID | 风险 | 等级 | 当前控制 | G1 前处理 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| R1 | 安装依赖和环境差异导致无法启动 | 高 | 启动脚本、健康接口 | 固定安装包和启动前检查 | 全新机器安装演练 |
| R2 | 升级或运行时迁移破坏数据库 | 阻断 | 启动时增量补列 | 版本化迁移、升级前备份、失败停止写入 | 旧版本数据库升级/回滚演练 |
| R3 | 数据库备份与文件目录不一致 | 阻断 | SQLite 备份与恢复脚本 | 统一备份清单和完整恢复检查 | 删除测试实例后全量恢复 |
| R4 | 会话令牌或模型密钥泄露 | 阻断 | 密钥环境变量、日志脱敏基础 | 会话/日志/导出/备份专项检查 | 搜索构建产物与日志、会话撤销测试 |
| R5 | 恶意或伪装文件消耗资源或绕过解析边界 | 阻断 | 大小、ZIP 路径和解析限制 | 文件签名、解析预算、异常隔离 | 伪装、压缩炸弹、超限用例 |
| R6 | 网页读取或工具请求访问不安全地址 | 阻断 | HTTPS、私网拒绝、白名单和超时 | 完整 SSRF 与重定向回归 | 私网、重定向、超大响应测试 |
| R7 | 模型调用和重试导致费用失控 | 高 | 单 Run 模型/工具预算 | 日/月硬预算、费用提醒和账单核对 | 并发、重试、预算耗尽测试 |
| R8 | 磁盘耗尽导致上传、数据库或 Run 失败 | 高 | 单文件限制 | 存储配额、剩余空间检查和告警 | 低磁盘模拟 |
| R9 | 模型、OCR 或网络依赖不可用但界面看似成功 | 高 | 部分错误事件和带限制回答 | 启动检查、明确降级、状态页 | 逐个关闭依赖演练 |
| R10 | 日志包含对话、资料、完整工具参数或密钥 | 阻断 | 部分安全摘要 | 结构化日志契约和敏感模式扫描 | 自动日志扫描 |
| R11 | 数据删除不完整或历史下载仍可访问 | 阻断 | 部分级联和当前权限复核 | 统一删除、下载再鉴权和备份边界 | 删除后 API/文件/历史引用测试 |
| R12 | 单进程故障中断长任务 | 中 | Run 状态、重试和重启恢复 | 个人版明确边界，补充故障说明 | 运行中重启演练 |
| R13 | Agent 质量样本不足却对外宣称提升 | 中 | 固定评测、反馈门槛 | 继续区分固定评测和真实样本 | 发布报告检查 |

阻断级风险未关闭时不得进入邀请测试。风险负责人暂由平台开发者兼部署者承担；开始邀请测试后，每项风险必须明确到实际维护人。

## 6. G1 准入清单

### 安装和首次使用

- 【】全新 macOS/Linux 环境在 30 分钟内完成安装。
- 【】首次启动能检查管理员、模型、OCR/文档依赖、目录权限和磁盘。
- 【】用户无需编辑代码即可完成首份资料上传和首次有效对话。

### 账号和安全

- 【】生产环境没有默认密码或默认可登录账号。
- 【】密码修改、账号恢复、退出全部设备和过期会话清理通过。
- 【】R2、R3、R4、R5、R6、R10、R11 等阻断风险关闭。

### 数据和恢复

- 【】对话、知识、产物、记忆和设置可导出。
- 【】账号/数据删除边界清楚，并通过数据库、文件、下载和引用验证。
- 【】完成一次全量备份和隔离恢复，关键表数量与文件清单一致。
- 【】升级失败不会破坏可恢复的上一版本数据。

### 费用和容量

- 【】单 Run、每日和每月模型预算由服务端执行。
- 【】知识、产物和磁盘容量有上限及接近上限提醒。
- 【】至少 10 个真实模型 Run 与供应商账单完成费用核对。

### 质量和运行

- 【】至少 10 个真实风格任务完成并记录成功、失败和边界。
- 【】前端检查、服务端完整回归、固定评测和安全专项测试通过。
- 【】模型不可用、数据库只读、磁盘不足、OCR 缺失和服务重启演练通过。
- 【】Run 失败可通过运行详情或安全日志定位，不需要查看用户私密正文。

### 试用运营

- 【】公开注册关闭，邀请为限时且单次使用。
- 【】试用用户确认数据位置、模型费用、备份责任和已知限制。
- 【】提供问题反馈、数据导出和停止使用流程。

## 7. 批次 A 结论

P46-0/P46-1 的规划基线已经建立，现有 Agent 核心、知识库、审批、审计和评测可以复用。当前不能进入 G1，主要阻断集中在可重复安装、版本化迁移、一致备份恢复、账号恢复、数据导出/删除、文件安全以及日/月费用控制。

下一开发批次严格进入 P46-B：安装、配置、账号和升级。不得同时启动公开注册、支付、团队协作或基础设施重构。
